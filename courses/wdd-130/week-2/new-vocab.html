New Vocabulary

Last week you were introduced to some of the most important terms you need to know about HTML and CSS. This week we want to highlight some new terms that are important for web development in general. Please work on memorizing these terms. You will be quizzed at the end of the week on these terms and other material from this week.

Assignment

  1. General Terms

    Make sure you have read, understood, and memorized the following terms:

    A (Anchor)
    The <a> tag defines a hyperlink, which is used to link from one page to another. This tag has also been called the anchor tag or back tag because it links (anchors) back to other resources on the internet.
    Body
    The <body> tag defines the document's body and contains all the contents of an HTML document, such as headings, paragraphs, images, hyperlinks, tables, lists, etc.
    CDN

    A distribution system on the Internet that accelerates the delivery of web pages, audio, video and other Internet-based content to users around the world

    When content is replicated throughout the country or the world, it is delivered to users with greater speed and reliability. The CDN network routes the user's request for content to the appropriate caching server based on the user's location.

    This summarized definition comes from the CDN definition entry by PCMAG. You should visit the link for the full entry and to learn more about CDN's.

    External
    In the context of web development external refers to an external file. Often we will link to external style sheets and script files. This allows developers to keep code files separate and easily manageable, web browsers will then treat all the separate files as a single file.
    Font Weight
    The font-weight property sets how thick or thin characters in text should be displayed.
    Head

    The <head> element is a container for metadata (data about data) and is placed between the <html> tag and the <body> tag at the top of your HTML document.

    The <head> element is a container for the following elements: <title>, <style>, <meta>, <link>, <script>, and <base>.

    Inline
    In the context of web development inline refers to styles or scripts added directly inline with an HTML element. Because this type of code is difficult to maintain it should be used sparingly or avoided altogether. Here are some examples of inline code: 
    <h1 style="color:blue;">A Blue Heading</h1>
<p style="color:red;">A Red Paragraph.</p>
<div onclick="runFun();">A div that reacts to being clicked on.</div>
    Internal
    In the context of web development internal refers to styles or scripts added inside (internal to) the file they are used in. Well this convenient for small style rules or scripts, it makes maintaining and updating code harder. External files should be used as much as possible.
    Link

    The <link> tag defines the relationship between the current document and an external resource; it is most often used to link to external style sheets.

    Meta

    The <meta> tag defines metadata about an HTML document. Metadata is data (information) about data. Metadata will not be displayed on the page, but is machine parsable.

    <meta> tags always go inside the <head> element, and are typically used to specify character set, page description, keywords, author of the document, and viewport settings.

    Style Sheet
    An external style sheet is used to define the style for many HTML pages at once. A CSS (style sheet) file should be saved with the .css file extension.
