Troubleshooting

Anyone who writes code or creates software will eventually master the art of troubleshooting. Often we will run into two questions when we write code: Why is this not working? and How is this working?. The first question (why) is what builds our troubleshooting skills and the second (how) is what helps us become troubleshooting masters. In this article we will share with you some of the common mistakes developers make and the troubleshooting skills to fix them.

Naming Conventions

How you name things matters. Well not always true, it's safe to assume anything you code or name is case sensitive. Here is an example of 3 different file names. They all have the same spelling but are considered different because of case sensitivity: 

hello.txt
Hello.txt
HeLlo.txt

Technically speaking CSS id and class names are case-insensitive but there are exceptions to this rule so it is best to stick to the same case.

When coding make sure to look up the correct naming conventions for the language your using. In previous versions of HTML for example, id's could not start with a number.

Correct File Extensions

How you save a file is just as important as how you name it. If your working with an HTML file but fail to save it with the .html extension, web browsers will not be able to load the file correctly. The same goes with all types of files. A common mistake new developers make is using the incorrect file extension for the type of file they are working with. For example saving a stylesheet file with the .html extension instead of the .css extension.

Correct File Locations

If a resource or page fails to load it often means the file is missing. On a website this often triggers a 404 error. This could mean you forgot to upload a file or your trying to access a file in the incorrect location. Make sure all the files you expect to be present actual are. Next check that your accessing or linking to the file in the correct location. The following video may help. If you still have issues do an internet search on absolute vs relative file paths.

Incorrect or Missing Closures

Another common mistake new developers make is using incorrect closures; or leaving the closure out altogether. In HTML for example we often have opening tags that require a closing tag (closure). Here is an example of an improperly and properly closed paragraph tag: 

<p>Improperly Closed<p>
<p>Properly Closed</p>

Improper closures can happen anywhere in your code. A common one is forgetting the semicolon ; or the closing curly bracket } in languages that require it like JavaScript or CSS rules and rule sets.

Not Validating Code

New web developers often forget to validate their code or ask for help to quickly. With modern tools code validation is an easy process every developer should become familiar with. Here is an example of the troubleshooting steps a web developer might take to find and fix an error:

  1. Make sure there are no red or yellow lines in your editor. Red lines or marks indicate the editor detected an error that needs fixing. Yellow is calling your attention to something but is not necessarily an error.
  2. If the file your working on is accessible in a web browser open the web browser inspector ctrl + shift + i (or right click and choose inspect) and see if the console tab is showing any errors. Do an internet search to find help on what this error means.
  3. Use a validation service that will detect errors and offer recommendations to you. For HTML files you can use the W3C Markup Validation Service and for CSS files you can use the W3C CSS Validation Service. There is also the ECMAScript Syntax Validator for JavaScript.
