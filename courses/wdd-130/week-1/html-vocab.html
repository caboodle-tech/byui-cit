HTML Vocabulary

Learning the vocabulary for HTML will be vital for your success in this class. Like any language it is impossible to learn everything all at once, you should however memorize the following core terms. You will have a hard time making sense of the instruction given in this course if you don't understand these terms.

Assignment

  1. Core Terms

    Make sure you have read, understood, and memorized the following terms:

    Tag
    HTML tags are key words surrounded by angle brackets. For example here is the opening body tag: 
    <body>
    and here is the closing /body tag: 
    </body>
    Not all tags need a matching closing tag.
    Element
    An HTML element usually consists of a start tag and end tag, with the content inserted in between: 
    <p>Hello!</p>
    Nesting
    Elements placed inside of other elements are called nested. Nested Elements are indented to make it easier to see which elements are inside of others. Well formatted code (good nesting) makes it easier for the developer to read their code, maintain it, and troubleshoot it. 
    <div>
    <p>Content.</p>
</div>
    Parent
    In nesting the parent is the element that contains other elements. 
    <div> <!-- Parent -->
    <p>Content.</p>
</div>
    Child
    In nesting a child element is contained inside of another element. 
    <div>
    <p>Content.</p> <!-- Child -->
</div>
    Attribute
    Attributes provide additional information about HTML elements. They are placed inside of the opening tag. The value of the attribute should be wrapped in quotes. Here is a paragraph tag with a title attribute: 
    <p title="paragraph">Idaho.</p>
    Semantics
    Semantics is the study of the meanings of words and phrases in a language. Semantic elements are elements with a meaning. Writing semantic HTML means using the html tag that best describes the content you are adding to your page. Here is an online tutorial you should study to learn more about semantic elements.
