Accessibility

User accessibility has been mentioned several times in this course, however given that this course is meant to be an introductory to web fundamentals, we have not explored this topic in great detail. In this study assignment you will receive a crash course in designing software for user accessibility. Please study the following resources keeping in mind that we do not expect you to understand or apply some of these advanced concepts in this course.

Assignment:

  1. HTML vs HTML5

    HTML5 semantic elements are useful for user accessibility but they are more a byproduct than an intention. HTML5 was designed to replace earlier versions of HTML and standardize web development globally. Now that HTML5 is the defacto standard for web development it has continued to evolve gaining more and more accessibility support. Please read the following article to learn more about the history of HTML5: HTML vs HTML5 - What Is the Difference?

  2. ARIA

    ARIA stands for: Accessible Rich Internet Applications. Many of the interactive web features you use on a daily basis would be impossible for assistive technology to operate and interact with without ARIA. For example, without ARIA an impaired user wanting to navigate the same websites you do on a daily basis, would need assistive technology specially adapted and designed for each website. ARIA allows developers to indicate with HTML and JavaScript how interactive elements should behave for users that can not interact with them in a standard way. Please review the following article introducing ARIA, note any questions you have, and ask your instructor about them in class: MDN Web Docs: ARIA

  3. What Visual Impairment Means for Your Software

    Well there are many types of disabilities and user impairments, you will complete your crash course in designing for user accessibility by reading an article about designing websites for visually impaired users. Please note any questions you have about this article so you can ask your instructor about them in class: Understanding What Visual Impairment Means for Your Software

  4. Complete

    There is nothing to turn in for this study assignment but you will be quizzed on this material later in the week.
